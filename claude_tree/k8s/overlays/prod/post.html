GitOps란 무엇인가?

GitOps는 Git을 단일 진실 공급원(Single Source of Truth)으로 사용하여 인프라와 애플리케이션을 선언적으로 관리하는 현대적인 배포 방식입니다. 모든 변경사항이 Git 커밋으로 추적되며, 자동화된 프로세스가 이를 운영 환경에 반영합니다.

💡 핵심 원칙
  • 모든 것을 Git에서 관리
  • 선언적 구성
  • 자동화된 배포
  • 지속적인 동기화

ArgoCD 소개

ArgoCD는 Kubernetes를 위한 선언적 GitOps 지속적 배포 도구입니다. Git 저장소의 매니페스트를 모니터링하고 클러스터 상태와 자동으로 동기화합니다.

주요 기능

  • 자동 동기화: Git 저장소의 변경사항을 자동으로 감지하고 배포
  • 상태 모니터링: 실시간으로 애플리케이션 상태 추적
  • 롤백: 이전 버전으로 쉽게 되돌리기
  • 멀티 클러스터: 여러 클러스터를 하나의 ArgoCD로 관리

ArgoCD 설치하기

ArgoCD를 Kubernetes 클러스터에 설치하는 방법입니다:

# ArgoCD 네임스페이스 생성
kubectl create namespace argocd

# ArgoCD 설치
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml

# ArgoCD CLI 설치 (macOS)
brew install argocd

# 초기 비밀번호 확인
kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d

Kustomize와 함께 사용하기

Kustomize를 사용하면 환경별로 다른 설정을 효율적으로 관리할 수 있습니다. ArgoCD는 Kustomize를 기본적으로 지원합니다. 

# base/kustomization.yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization

resources:
  - deployment.yaml
  - service.yaml

# overlays/dev/kustomization.yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization

bases:
  - ../../base

namePrefix: dev-
namespace: dev

replicas:
  - name: app
    count: 1

Application 생성하기

ArgoCD Application을 정의하여 배포를 자동화합니다:

apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: blog-board-dev
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://github.com/your-org/your-repo
    targetRevision: HEAD
    path: k8s/overlays/dev
  destination:
    server: https://kubernetes.default.svc
    namespace: blog-app-dev
  syncPolicy:
    automated:
      prune: true
      selfHeal: true

베스트 프랙티스

1. 환경 분리

Dev, Staging, Production 환경을 별도의 오버레이로 관리하세요.

2. 자동 동기화 설정

syncPolicy.automated를 설정하여 Git 변경사항을 자동으로 반영합니다. prune과 selfHeal을 활성화하면 더욱 강력합니다.

3. Health Check

리소스의 상태를 정확히 확인하기 위해 적절한 Health Check를 설정하세요.

⚠️ 주의사항

Production 환경에서는 자동 동기화를 신중하게 설정하세요. 수동 승인 프로세스를 고려할 수 있습니다.

결론

ArgoCD를 활용한 GitOps는 배포 프로세스를 투명하고 추적 가능하게 만듭니다. Git의 버전 관리 기능과 결합하여 안전하고 효율적인 배포 파이프라인을 구축할 수 있습니다.
