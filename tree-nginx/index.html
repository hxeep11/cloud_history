nginx-tree 포털
OKE / Kubernetes 리소스 뷰 요약 페이지 (static HTML)
Cluster: dev-oke-cluster
Namespace: portal
리소스 트리
nginx가 서빙하는 정적 페이지용 트리 구조입니다.
실제 리소스 구조에 맞춰 자유롭게 노드를 추가하시면 됩니다.
  • cluster dev-oke-cluster
    • ns portal namespace
      • deploy portal-nginx nginx
        • svc portal-nginx-svc ClusterIP
        • cm portal-nginx-index index.html
      • deploy tree-nginx static
        • svc tree-nginx-svc NodePort
    • ns argocd cd
      • ing argocd-server Ingress
포털 개요
이 페이지는 Helm + nginx(staticSiteConfigmap) 조합으로 서빙되는 정적 HTML입니다.

구조 설명

- 상단 헤더: 클러스터 / 네임스페이스 정보 표시
- 좌측 트리: cluster → namespace → deployment → service / configmap 등 계층 구조
- 우측 패널: 간단한 설명, 향후에는 yml snippet, 링크 등을 붙여도 됨

index.html 배포 방식

1) 이 파일을 index.html 이름으로 저장합니다.
2) 아래 명령으로 ConfigMap 생성:
   kubectl create configmap portal-nginx-index \
     --from-file=index.html \
     -n portal

3) bitnami/nginx values 에서:
   staticSiteConfigmap: "portal-nginx-index"

4) helm upgrade:
   helm upgrade portal-nginx bitnami/nginx \
     -n portal \
     -f values-portal-nginx.yaml

메모

- 실제 리소스 이름(클러스터명, 네임스페이스명, 서비스명 등)을
  운영 환경에 맞게만 바꿔 주시면 됩니다.
- 필요하면 이 페이지에 각 서비스의 대시보드 링크, Grafana, ArgoCD,
  Kiali 등으로 가는 하이퍼링크를 추가해도 좋습니다.
